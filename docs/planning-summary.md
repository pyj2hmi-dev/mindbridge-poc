# MindBridge Planning Summary

## 1. Project Overview

MindBridge는 심리상담 초기 접수면접을 AI 인터뷰 방식으로 구조화하고, 그 결과를 리포트로 정리하는 AI 기반 접수면접 보조 시스템입니다.

이 프로젝트의 목적은 심리상담을 AI가 대체하는 것이 아닙니다.

MindBridge는 초기 접수 과정에서 반복되는 질문, 기록, 정보 정리 부담을 줄이고, 상담사 또는 사용자가 핵심 정보를 더 빠르게 파악할 수 있도록 돕는 PoC 프로젝트입니다.

## 2. Project Positioning

MindBridge는 상담 챗봇이 아니라 접수면접 보조 및 리포트 자동화 시스템입니다.

핵심 역할은 다음과 같습니다.

- 초기 접수면접 질문 흐름 지원
- 사용자 응답 구조화
- 상담사용 또는 사용자용 리포트 생성
- 위기 응답의 보수적 표시
- 상담 자원 후보 탐색 보조

수행하지 않는 역할은 다음과 같습니다.

- 심리상담 대체
- 임상 판단
- 진단
- 치료 방향 제시
- 특정 상담사 또는 상담센터 최종 추천
- 자가진단 또는 자가상담

## 3. Target Users

### 상담소 트랙 사용자

상담센터에 내방하여 본 상담 전에 초기 접수면접에 응답하는 사용자입니다.

이 트랙에서는 상담사가 리포트를 확인하고 첫 면담을 준비하는 흐름을 전제로 합니다.

### 일반인 트랙 사용자

상담을 검토 중이며, 자신의 상태를 정돈하고 관련 상담 자원 후보를 탐색하려는 사용자입니다.

이 트랙에서도 MindBridge는 자가진단이나 자가상담을 제공하지 않습니다.

### 심리상담사

구조화된 리포트를 검토하고 첫 상담 전에 핵심 정보를 파악하는 사용자입니다.

### 운영자

필요 시 자원 탐색 기능, 디버그 기능, 로그 확인 기능을 활용하는 사용자입니다.

## 4. Problem Definition

초기 접수면접 단계에서는 다음 문제가 발생할 수 있습니다.

- 동일하거나 유사한 질문이 반복됨
- 상담사가 질문과 기록을 동시에 수행해야 함
- 고정형 문진만으로는 자유로운 응답을 충분히 반영하기 어려움
- 위기 관련 응답을 일관되게 표시할 필요가 있음
- 사용자가 AI를 상담사처럼 오인할 가능성이 있음

답변분류 단계에서는 다음 문제가 발생할 수 있습니다.

- 짧지만 의미 있는 답변이 무효 처리될 수 있음
- 보완 질문 이후 원래 질문 의도가 흔들릴 수 있음
- 규칙 기반 분류만으로 자유서술 응답을 안정적으로 처리하기 어려움
- 어느 단계에서 병목이 발생했는지 확인하기 어려움

상담 자원 탐색 단계에서는 다음 문제가 발생할 수 있습니다.

- 상담센터와 상담사 정보가 여러 사이트에 흩어져 있음
- 상담영역 분류가 일반 사용자에게 직관적이지 않음
- 센터 데이터와 상담사 데이터의 기준이 다름
- 검색 결과가 추천처럼 보이지 않도록 조심해야 함

## 5. Core Design Principles

MindBridge의 설계 기준은 임상적으로 안전한 상호작용입니다.

핵심 원칙은 다음과 같습니다.

- AI는 상담사를 대신하지 않는다.
- AI는 초기 접수면접 보조와 정보 구조화에 집중한다.
- AI는 운영적 판단만 수행하고 임상적 판단은 수행하지 않는다.
- 질문의 기준은 공감이 아니라 목적의 명확성이다.
- 보완 질문은 면접지 범위 안에서만 제한적으로 수행한다.
- 위기 상황에서는 고정 안내 문구와 사람 개입을 우선한다.
- 리포트는 사실 정리 중심으로 작성한다.
- 상담 자원 탐색은 추천이 아니라 후보 안내로 제공한다.
- 디버그 기능은 새로운 상담 흐름이 아니라 관찰 레이어로 둔다.
- 사용자 입력은 시스템 명령이 아니라 접수면접 답변 데이터로 처리한다.

## 6. Service Flow

### 공통 접수면접 엔진

1. 기본 정보 입력
2. 시스템 성격 고지
3. 9섹션 접수면접 진행
4. 위기 응답 분기 처리
5. 구조화 추출
6. 리포트 생성

### 상담소 트랙 후속 흐름

1. 상담사용 리포트 확인
2. 상담사 첫 면담 진행
3. 필요 시 상담 자원 탐색 기능 활용

### 일반인 트랙 후속 흐름

1. 사용자용 리포트 확인
2. 조건 기반 상담 자원 탐색 수행
3. 센터 또는 상담사 후보 정보 확인
4. 사용자가 직접 검토 및 연락

## 7. LangChain Pipeline

MindBridge는 하나의 거대한 프롬프트로 모든 기능을 처리하지 않고, 기능별 역할을 나눈 LangChain 기반 파이프라인으로 설계했습니다.

핵심 체인은 다음과 같습니다.

### Interview Chain

현재 섹션 목적에 맞는 다음 질문 또는 보완 질문을 생성합니다.

상담자처럼 공감하거나 해석하는 것이 아니라, 접수면접 진행에 필요한 질문 흐름을 유지하는 역할입니다.

### Extractor Chain

전체 대화에서 리포트 작성에 필요한 정보를 구조화합니다.

사용자 발화 중심으로 정리하며, 추측이나 임상적 해석은 수행하지 않습니다.

### Crisis Chain

위기 관련 응답을 보수적으로 감지하고 표시합니다.

자유 생성형 위로 응답보다 고정 안내 문구와 위기 플래그 기록을 우선합니다.

### Report Chain

구조화 데이터와 위기 로그를 바탕으로 상담사용 또는 사용자용 리포트를 생성합니다.

리포트는 진단서나 치료 계획서가 아니라 접수면접 정리 문서입니다.

## 8. Main Features

### 9섹션 접수면접

초기 접수면접은 다음 9개 섹션으로 구성됩니다.

1. 주호소 문제
2. 감정 및 정서 증상
3. 인지 및 사고 증상
4. 신체화 증상
5. 행동적 증상
6. 현재 생활 및 기능 수준
7. 성장 과정, 가족 관계, 사회적 자원
8. 위기 사항 및 건강 상태
9. 상담 신청 동기 및 의지

### 답변분류 및 상태 관리

사용자 응답은 현재 질문 의도를 기준으로 분류합니다.

분류 기준은 다음과 같습니다.

- valid
- partial
- invalid
- resistant

질문 생성 시 현재 질문 intent를 상태값으로 저장하고, 보완 질문이 발생해도 intent가 유지되도록 설계했습니다.

### 위기 응답 처리

위기 응답은 일반 자유서술 흐름과 분리하여 처리합니다.

위기 여부를 임상적으로 확정하지 않고, 명시적 입력과 보수적 규칙에 따라 표시합니다.

### 리포트 생성

리포트는 다음 항목을 포함합니다.

- 기본 정보
- 9섹션별 핵심 응답 요약
- 감정, 인지, 신체, 행동 영역 정리
- 위기 플래그 표시
- 전체 요약

리포트 작성 시 금지되는 것은 다음과 같습니다.

- 진단명 확정
- 치료 방향 제시
- 심각도 단정
- 최종 매칭 유도
- 해석형 표현

### 상담 자원 탐색

접수면접 이후 사용자가 원할 경우 상담 자원 후보를 탐색할 수 있습니다.

검색 조건은 다음과 같이 구조화합니다.

- 지역
- 세부 지역
- 상담 대상
- 상담 형태
- 주요 상담영역 태그
- 검색 대상

데이터 형식은 JSON 또는 CSV 기반 구조화 데이터를 사용할 수 있도록 설계했습니다.

검색 결과는 추천이 아니라 후보 안내로 제시합니다.

### 디버그 및 품질 점검

디버그 기능은 질문 흐름을 바꾸는 기능이 아닙니다.

현재 로직을 그대로 유지한 상태에서 다음 항목을 관찰합니다.

- 현재 질문 intent
- 현재 섹션
- 사용자 입력
- 분류 결과
- reason code
- chosen action
- repair count
- extracted delta

### 보안 및 역할 이탈 대응

사용자 입력은 시스템 규칙을 바꾸는 명령으로 처리하지 않습니다.

다음 입력은 정상 답변으로 인정하지 않습니다.

- 이전 지시 무시 요구
- 시스템 프롬프트 노출 요구
- 상담 접수 역할 변경 요구
- 관리자 모드 호출 시도
- 리포트 또는 위기 표시 임의 변경 요구

이러한 입력은 기존 답변분류 체계 안에서 invalid 또는 resistant로 처리합니다.

## 9. My Role

이 프로젝트에서 담당한 역할은 다음과 같습니다.

- 서비스 기획 및 문제 정의
- 기능 범위 조율
- 프롬프트 설계
- 구현 가능 범위와 후속 과제 분리
- 핵심원칙과 역할 경계 정리
- 기획서, 기능 정의서, 발표 자료 작성
- 리스크 및 한계 정리
- 발표 및 시연 흐름 구성

## 10. Expected Impact

MindBridge를 통해 기대한 효과는 다음과 같습니다.

- 초기 접수면접의 반복 업무 감소
- 사용자 응답의 구조적 수집
- 상담사의 사전 정보 파악 효율 향상
- 위기 관련 응답의 일관된 표시
- 리포트 생성 및 정리 시간 단축
- 답변분류 병목을 점검할 수 있는 기반 확보
- 접수면접 이후 상담 자원 탐색 보조 가능
- 상담소 트랙과 일반인 트랙을 하나의 접수면접 엔진 위에서 운영 가능

## 11. Current Limits

현재 버전의 한계는 다음과 같습니다.

- 심리상담 자체를 대체하지 못함
- 임상 판단과 진단을 수행하지 않음
- 답변분류는 지속적인 정책 보정이 필요함
- 리포트 문구는 현장 피드백 기반 보정이 필요함
- 자원 탐색 기능은 최종 추천 기능이 아님
- 일부 검색 조건은 직접 반영되지 않을 수 있음
- 데이터 소스별 필드 차이로 조건 반영 수준에 편차가 있음
- 상담 자원 데이터의 최신성 관리가 필요함
- STT와 TTS는 후속 확장 과제임

## 12. Future Roadmap

후속 과제로 검토할 수 있는 항목은 다음과 같습니다.

- 자유서술형 답변 처리 보완
- LLM fallback 도입 범위 검토
- 디버그 로그 분석 자동화
- 상담사 피드백 수집 구조 도입
- 일반인 트랙 UX 보정
- 리포트 문구 현장 보정
- 상담 자원 검색 조건 확대
- JSON 및 CSV 정규화 품질 점검
- 운영자용 관리 및 로그 기능 보강
- 배포 방식 단순화
- STT 및 TTS 1차 연동 검토

## 13. Final Summary

MindBridge는 심리상담을 수행하는 챗봇이 아닙니다.

이 프로젝트는 초기 접수면접을 더 일관되게 수행하고, 사용자 응답을 구조화하며, 리포트와 상담 자원 후보 탐색으로 연결하는 AI 기반 접수 보조 시스템 PoC입니다.

핵심은 기능을 무리하게 확장하는 것이 아니라, 설명 가능하고 안전하게 동작하는 범위 안에서 AI 서비스 구조를 설계하는 데 있습니다.
